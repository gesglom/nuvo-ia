# Agent Runtime Observability

## Objetivo
Asegurar operación observable con métricas y diagnóstico de fallos por agente y proveedor.

## Checklist
- Revisar `/api/metrics` y distribución por agente.
- Verificar latencia promedio y ratio de fallos.
- Revisar `self_improvement` para patrones de error.
- Definir SLOs mínimos (éxito, latencia).
